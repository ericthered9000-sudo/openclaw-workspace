# Competitor Analysis

This skill provides a systematic workflow for researching and analyzing competitors in any app/product market.

## Workflow

### Step 1: Define the Market

Ask the user for:
- Product category (health, productivity, finance, etc.)
- Target audience (seniors, professionals, parents, etc.)
- Key features/functionality
- Geographic markets (US, EU, Global)
- Platforms (iOS, Android, Web)

### Step 2: Identify Competitors

Search for competitors using multiple methods:

**App Store Searches:**
- Google Play Store search for category keywords
- Apple App Store search for category keywords
- Search for similar feature sets

**Web Searches:**
- "[category] app"
- "[feature] app for [audience]"
- "[competitor] alternatives"
- "best [category] apps 2024/2025"

**Review Sites:**
- Product Hunt
- G2, Capterra (for B2B)
- Tech blogs and review sites

Document:
- Direct competitors (same features, same audience)
- Indirect competitors (same audience, different features)
- Adjacent competitors (similar features, different audience)

### Step 3: Analyze Each Competitor

For each competitor, research:

**Product:**
- Core features
- Unique features
- User experience quality
- Platform availability

**Pricing:**
- Free tier features
- Paid tier pricing (monthly/annual)
- Enterprise pricing
- In-app purchases

**Market Position:**
- Download counts (if available)
- User ratings
- Review sentiment
- Last update date
- Company size/funding

**Strengths & Weaknesses:**
- What they do well
- What users complain about
- Feature gaps
- UX issues

### Step 4: Analyze Market Size

Research:
- Total addressable market (TAM)
- Market growth rate
- Key trends
- Recent acquisitions
- Funding activity

Sources:
- Industry reports (often behind paywalls, search summaries)
- News articles about market size
- Similar companies' investor presentations
- App store category rankings

### Step 5: Identify Gaps

Based on competitor analysis, identify:
- Missing features across all competitors
- Underserved user segments
- Pricing gaps
- UX improvements
- Platform gaps (iOS-only but no Android, etc.)

### Step 6: Output Analysis Report

Create a structured report:

```markdown
# Competitor Analysis: [CATEGORY]

## Executive Summary
- Market size: [X]
- Key competitors: [list]
- Market gap: [opportunity]

## Competitor Matrix

| Competitor | Pricing | Key Features | Users | Rating |
|------------|---------|-------------|-------|--------|
| ... | ... | ... | ... | ... |

## Competitor Profiles

### [Competitor 1]
- **Description:** [brief description]
- **Pricing:** [free tier, paid tiers]
- **Features:** [key features]
- **Strengths:** [what they do well]
- **Weaknesses:** [user complaints, gaps]
- **Users:** [download count, user base]
- **Last Updated:** [date]

[Repeat for each competitor]

## Market Analysis

### Market Size
- TAM: $X billion
- Growth Rate: X% CAGR
- Key Trends: [list]

### Pricing Landscape
- Budget tier: $X-X/month
- Mid tier: $X-X/month
- Premium tier: $X+/month
- Average: $X/month

## Market Gaps

| Gap | Competitors Missing It | Opportunity |
|-----|------------------------|-------------|
| ... | ... | ... |

## Recommendations

1. [Primary recommendation]
2. [Secondary recommendation]
3. [Risk mitigation]

## Sources
- [List of sources]
```

Save report to: `research/[CATEGORY]-competitor-analysis-[DATE].md`

## Tips

- **Look at 1-star reviews** — These reveal pain points and gaps
- **Check update frequency** — Abandoned apps signal opportunity
- **Compare feature parity** — What's table stakes vs differentiator
- **Note pricing tiers** — Where are competitors leaving money on the table?
- **Identify copycats** — Many markets have obvious clones with market share
- **Check company backgrounds** — Funded startups may pivot or shut down
