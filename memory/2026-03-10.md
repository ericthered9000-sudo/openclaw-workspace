## Workflows
### When Things Work (at work, WhatsApp, lunch):
- Clear goal, time limited
- One fix, execute, verify, done
- No git complications

### When Things Break (at home, computer):
- Multiple contexts, tabs, sessions
- Git state confusion
- Reconstructing instead of fixing

### Better Workflow:
1. Extract exact working code from APK
2. Commit with clear message
3. User says what to fix
4. Fix ONLY that thing
5. Show BEFORE deploying
6. User approves
7. Deploy
8. COMMIT immediately
