# March 9, 2026 - Sunday Session

## HomeBeacon App Fixes

### Issues Found and Fixed

#### 1. Vercel Routing Problem
**Problem:** The `vercel.json` used legacy `routes` config with a catch-all `{ "src": "/(.*)", "dest": "/index.html" }` that intercepted ALL requests - including static assets like JS and CSS files. The browser would request `/assets/index-xxx.js` and get HTML instead of JavaScript.

**Fix:** Changed from `routes` to `rewrites`:
```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "vite",
  "rewrites": [
    { "source": "/privacy", "destination": "/privacy.html" },
    { "source": "/terms", "destination": "/terms.html" },
    { "source": "/support", "destination": "/support.html" },
    { "source": "/app", "destination": "/index.html" },
    { "source": "/app/:path*", "destination": "/index.html" }
  ]
}
```

#### 2. API URL Environment Variable Had Trailing Newline
**Problem:** `VITE_API_URL` environment variable on Vercel had a newline character at the end:
```javascript
const Te="https://web-production-332e5.up.railway.app\n";
```
This broke ALL API calls because URLs with embedded newlines don't work.

**Fix:** Removed and re-added the environment variable:
```bash
vercel env rm VITE_API_URL production --yes
echo -n "https://web-production-332e5.up.railway.app" | vercel env add VITE_API_URL production
```

#### 3. Deployment Directory Confusion
**Problem:** The workspace root `.vercel` folder was overriding the frontend's project config, causing deployments to go to the wrong project.

**Fix:** Removed workspace `.vercel` folder before deploying:
```bash
rm -rf /home/james/.openclaw/workspace/.vercel
cd /home/james/wellness-check-code/frontend && vercel --prod --yes
```

## Lessons Learned

1. **Vercel `routes` vs `rewrites`:** Use `rewrites` for SPAs - `routes` with catch-all patterns will intercept static assets too.

2. **Environment variables with newlines:** When setting Vercel env vars via CLI, use `echo -n` to avoid trailing newlines, or use `printf` to be explicit.

3. **Multiple .vercel folders:** If you have nested projects, each needs its own `.vercel/project.json` and you must deploy from the correct directory.

4. **Debugging approach:** Checking the actual deployed JavaScript bundle (`curl` the JS file) revealed the corrupted API URL - the HTML response was the key clue that routing was broken.
